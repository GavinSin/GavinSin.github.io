Difference between Web Service and API

2020-04-15
API • Web Service • Definitions

There is a common misconception that API and Web Service are the same. These are especially true when people use both of the words interchangeably. One key thing to keep in mind:

Comparing API and Web Service is like comparing Fruit and Apple. While all apples are a kind of fruit, not all fruits are apples.

Definitions

API is an acronym for Application Programming Interface that allows two applications to interact with each other without any user interaction.

Web service, on the other hand, is a software system designed to support interoperable machine-to-machine interaction over a network. It has an interface described in a machine-processable format (specifically WSDL).

As you can see both of them are sets of protocol serve as a means of communication. To further understand their differences, we need to first classify different types of API by use cases.

Types of API

Source Code APIs:

These APIs include interfaces, Library based APIs, and Class-Based APIs

Hardware APIs:

Communication through hardware like reading temperature, flow, pressure, moisture, information like model number, etc is done with the help of Hardware APIs. However, these APIs are independent of browser compatibility.

Object Remoting APIs:

These APIs use remoting protocols like CORBA.

Class-based APIs:

Data and functionality are organized around classes just like in object-oriented languages in these APIs.

Web Service APIs:

It is one of the commonly used API and is used by REST, SOAP and XML-RPC for communication.

As you can see API is not just used for the web but can also be used by a standalone computer such as Linux kernelâ€“user space API, which allows programs in user space to access system resources and services of the Linux kernel.

The reason why some people is confused about the two, or at least for my case, is that I mostly interact with the web development side of thing and less with the hardware and low-level programming stuff. And most of the APIs I touched on are Web Service APIs, specifically REST APIs, so I instinctively assumed that API = Web Service, which is simply not true.

Key Difference

  • Web service is used for REST, SOAP and XML-RPC for communication while API is used for any style of communication.
  • APIs > Web Service APIs > REST APIs
  • Web service requires a network, while APIs can be both online or offline.
